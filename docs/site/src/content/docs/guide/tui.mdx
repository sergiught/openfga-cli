Run `ofga` with no arguments to launch the interactive playground. It's a keyboard- **and mouse**-driven cockpit for the whole OpenFGA surface.

The playground uses the same resolved profile as CLI commands: `ofga --profile
staging playground` opens staging without changing the saved default. Profile,
store, and model switches made inside the playground are reflected immediately
in its footer and subsequent actions.

**Sections** (switch with `tab`, the number keys `1`вҖ“`9`, `ctrl+k` for the command palette, or **click a tab**): Profiles В· Stores В· Model В· Tuples В· Changes В· Tuple Queries В· Assertions В· API Logs В· Tests.

**Highlights**

- рҹҺЁ **Model graph**: the authorization model rendered as a colored tree of types, relations, and inherited (tuple-to-userset) paths.
- рҹ”Һ **Query + resolution tree**: run `check`/`list-objects`/`list-users`/`list-relations` and expand *why* a decision was made.
- вңҚпёҸ **Inline editing**: add/delete tuples, edit assertions, and edit the model DSL, with **inline validation** as you type.
- рҹ–ұ **Full mouse support**: wheel-scroll the graph and lists, click tabs and list rows, click the footer keycaps as buttons, and click outside a dialog to dismiss it.
- рҹҺӯ **Themes**: `aurora`, `catppuccin`, `charm`, `dracula`, `gruvbox`, `nord`, `tokyonight`, and a `mono` (NO_COLOR-friendly) theme.

**Keys:** press `?` at any time for the full, context-aware keybinding overlay.

> The TUI only launches on an interactive terminal. In a pipe or CI, bare `ofga` prints help instead of hanging.
