Create or update a connection profile and make it active. On a terminal it runs a guided tour: it collects the API URL and authentication, tests the connection, and lets you pick a store and model from the server. Non-interactively it uses the flags and defaults, so it is safe in CI.

## Usage

```sh
ofga init [profile] [flags]
```

## Examples

```sh
  ofga init
  ofga init prod --api-url https://fga.example.com --token-stdin < token.txt
  ofga init prod --api-url https://fga.example.com --store-id 01STORE --model-id 01MODEL
```

## Flags

| Flag | Description | Default |
| --- | --- | --- |
| `--api-url` | API URL to save in the profile (default http://localhost:8080) |   |
| `-f, --force` | overwrite an existing profile without prompting | `false` |
| `--model-id` | authorization model ID to save in the profile |   |
| `--store-id` | store ID to save in the profile |   |
| `--token-stdin` | read the API token from stdin | `false` |
