Create a named connection profile. The auth method defaults to a bearer
token when --token* is given, otherwise none. For OAuth flows pass
--auth-method client_credentials or private_key_jwt with their fields.

## Usage

```sh
ofga profiles add <profile> [flags]
```

## Examples

```sh
  ofga profiles add dev --api-url http://localhost:8080 --use
  ofga profiles add prod --api-url https://fga.example.com --token-stdin < token.txt
  ofga profiles add ci --auth-method client_credentials \
    --client-id abc --client-secret-stdin --token-url https://issuer/oauth/token \
    --audience https://api.fga.example.com
```

## Flags

| Flag | Description | Default |
| --- | --- | --- |
| `--api-audience` | API audience (private_key_jwt) |   |
| `--api-url` | API URL (default `http://localhost:8080`) |   |
| `--audience` | OAuth audience |   |
| `--auth-method` | none \| api_token \| client_credentials \| private_key_jwt |   |
| `--client-id` | OAuth client ID |   |
| `--client-secret-file` | read the client secret from a file |   |
| `--client-secret-stdin` | read the client secret from stdin | `false` |
| `--key-file` | PEM signing key path (private_key_jwt) |   |
| `--key-id` | signing key ID (private_key_jwt) |   |
| `--model-id` | authorization model ID to save in the profile |   |
| `--scopes` | OAuth scopes (comma-separated) | `[]` |
| `--signing-method` | JWT signing method, e.g. RS256 (private_key_jwt) |   |
| `--store-id` | store ID to save in the profile |   |
| `--token-file` | read the API token from a file |   |
| `--token-stdin` | read the API token from stdin | `false` |
| `--token-url` | OAuth token endpoint URL |   |
| `--use` | switch to this profile after creating it | `false` |
