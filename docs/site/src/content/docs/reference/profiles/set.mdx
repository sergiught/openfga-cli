Set a field on the active profile (or --profile).

Connection: api_url, store_id, model_id.
Auth: auth_method (none|api_token|client_credentials|private_key_jwt), token,
client_id, client_secret, token_url, audience, api_audience, key_file,
private_key, signing_method, key_id, scopes (space-separated).

For secrets (token, client_secret, private_key) prefer --value-file or
--value-stdin so the value never appears in `ps` output or your shell
history. token, client_secret, and private_key are stored in the OS keyring.

## Usage

```sh
ofga profiles set <key> [value] [flags]
```

## Examples

```sh
  ofga profiles set api_url http://localhost:8080
  ofga profiles set auth_method api_token
  ofga profiles set token --value-stdin < token.txt
  ofga profiles set client_secret --value-file ./secret
  ofga profiles set private_key --value-file ./key.pem
```

## Flags

| Flag | Description | Default |
| --- | --- | --- |
| `--value-file` | read the value from a file instead of an argument |   |
| `--value-stdin` | read the value from stdin instead of an argument | `false` |
