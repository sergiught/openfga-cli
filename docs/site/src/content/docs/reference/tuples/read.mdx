Read tuples from the store. Use --user, --relation and --object to filter; all are optional. By default all matching tuples are returned (the CLI auto-pages); --max-results (alias --limit) caps the total returned and stops paging once reached. --page-size only tunes the per-request page.

## Usage

```sh
ofga tuples read [flags]
```

## Examples

```sh
  ofga tuples read
  ofga tuples read --object document:roadmap
  ofga tuples read --max-results 100
```

## Flags

| Flag | Description | Default |
| --- | --- | --- |
| `--limit` | alias for --max-results | `0` |
| `--max-results` | cap the total number of tuples returned (0 = unbounded) | `0` |
| `--object` | filter by object |   |
| `--page-size` | per-request page size (0 = server default; not a total cap) | `50` |
| `--relation` | filter by relation |   |
| `--user` | filter by user |   |
